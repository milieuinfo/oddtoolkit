\begin{abstract}
Linked open data remains difficult to operationalise for software teams when OWL-based models are documented primarily for ontology engineers rather than for implementers. We present \emph{ODDToolkit}, an open-source Ontology Driven Design toolkit that generates developer-facing artefacts from three inputs: an OWL ontology, a SKOS concept scheme that provides business-specific names, and a configuration file for developer-specific design implementations. From one enriched intermediate model, the toolkit emits SQL DDL, Java and TypeScript models, SHACL shapes, Mermaid diagrams, and Bikeshed documentation. The design goal is not fully automatic translation of arbitrary vocabularies, but repeatable regeneration of implementation artefacts as an ontology evolves inside a governed development workflow. We summarise the pipeline and report its use in a Flemish government environmental reporting project, where the generated artefacts supported implementation, documentation, and validation around a shared ontology.
\end{abstract}
